# HN TechPulse Roadmap

> 最后更新：2026-05-10

## 产品定位

HN TechPulse 是一个从 Hacker News 自动生成中文科技资讯视频的内容产品。它的目标不是把 HN 热帖简单搬运成摘要，而是做成一条有筛选、有判断、有节奏的科技观察栏目。

目标用户：
- 关注 AI、开发工具、开源、基础设施和创业趋势的中文技术受众
- 没时间每天刷 HN，但希望快速知道“今天技术圈在讨论什么、为什么重要、社区怎么分歧”
- 希望从 HN 评论里看到一线开发者和从业者的真实反馈

核心价值：
- **节省时间**：把当天高价值讨论压缩成可观看的视频
- **提供判断**：不只复述标题，还给出编辑角度和重要性解释
- **保留社区温度**：尽量呈现 HN 的真实分歧、观点分布和代表评论
- **形成栏目感**：让观众一眼知道这是 HN TechPulse，而不是通用 AI 摘要视频

## 当前实现状态

已经落地的能力：
- [x] HN 热帖与评论抓取
- [x] LLM 脚本生成
- [x] TTS 语音合成
- [x] Remotion 视频渲染
- [x] Dashboard、Event、Atmosphere、Quote、Closing 等基础卡片
- [x] Story 三段式结构雏形：事件 - 氛围 - 社区原声
- [x] 争议信号、观点分布、关键词、高质量评论等数据入口
- [x] 统一中文 UI 文案与基础文本截断策略
- [x] 开场与仪表板已经从“热榜搬运”转向“今日信号导览”
- [x] Story 卡片已经补齐 `editor_angle`、`why_it_matters`、`next_watch` 等产品化字段
- [x] Quote 卡片开始呈现观点对照，而不是随便抽引语
- [x] 字幕模式已区分标题卡/正文卡，减少视觉干扰
- [x] 缓存校验会检查 story 结构里是否具备产品化叙事字段

当前仍然存在的主要问题：
- 开场、章节标识和整体节奏还可以更像正式栏目，而不是“已经好看但还没完全定型”的成片
- Dashboard 的导览感已经有了，但全局品牌识别还不够强
- 争议指数和数据来源说明已经有基础展示，但对新观众的解释力还不够
- 自动化内容的质量护栏已经存在，但还需要更完整的渲染前检查和降级策略
- 表现力增强功能还没系统展开，仍以信息清晰优先

## 北极星目标

让观众在前 10 秒知道今天值得看什么，并在每条 story 的 20 秒内理解三件事：

1. 发生了什么
2. 为什么重要
3. 社区真正分歧在哪里

## Milestone 1：可读性与基础观感

目标：先把现有视频变得更干净、更稳定、更容易看懂。

状态：已完成
优先级：P0

### 已完成项

- [x] **统一 UI 语言**
  - 核心界面已统一为中文文案
  - Dashboard、Event、Quote、Atmosphere 等卡片的主要文案已统一

- [x] **简化 EventCard 信息层级**
  - 主标题、摘要、重要性、下一步观察等信息开始分层展示
  - 标题和正文都加入了更严格的长度控制
  - 原文标题和中文角度之间有了更清晰的层级关系

- [x] **调整字幕权重**
  - 标题卡和结尾卡使用更轻量的字幕模式
  - 正文字幕更短、更克制，不再压过画面

- [x] **修正可见乱码风险**
  - 渲染侧已经引入统一的文本清洗、HTML 转义处理和基础乱码检测
  - 中文文案与提示词的乱码问题已优先修复

- [x] **增加文本长度限制**
  - 标题、摘要、关键词、评论引述都有更明确的截断和降级策略
  - 超长内容优先摘要，不直接硬截断核心语义

### 验收标准

- 任意暂停一张事件卡，3 秒内能看懂核心信息
- 字幕不遮挡主标题，不抢画面
- 中文 UI 文案风格一致
- 长标题和长关键词不会撑坏布局
- 渲染样片中没有明显乱码

## Milestone 2：内容产品化

目标：让视频从“摘要集合”升级为“有编辑判断的栏目”。

状态：已完成
优先级：P0

### 已完成项

- [x] **重写开场钩子**
  - 开场已从品牌介绍转向“今天看点”
  - 开场会优先引用今天的具体技术信号，而不是只说“今天 HN 热门”

- [x] **固定 story 四段式叙事**
  - `发生了什么`
  - `为什么重要`
  - `社区怎么说`
  - `接下来观察什么`

- [x] **引入编辑角度字段**
  - `editor_angle`、`why_it_matters`、`next_watch`、`category` 已进入 story 结构和渲染链路
  - 这些字段已能影响开场、Dashboard 和 EventCard 的呈现

- [x] **Dashboard 改成导览页**
  - 不再只是密集榜单
  - 已可突出“今天先重点看哪几条”

- [x] **代表评论改成观点对照**
  - QuoteCard 已从“高质量引文展示”升级为“支持 / 质疑 / 中立”的观点对照
  - 更能体现社区分歧，而不是随机摘句

### 验收标准

- 开场 3 秒内出现具体议题，而不是只有品牌名
- 每条 story 都能回答“为什么值得关注”
- Dashboard 看起来像视频导览，而不是静态表格
- QuoteCard 能看出社区立场差异

## Milestone 3：栏目品牌与观看节奏

目标：形成稳定、可复用、可识别的栏目包装。

状态：进行中
优先级：P1

### 目前已具备的基础

- [x] **建立基础视觉系统**
  - 深色科技感主色系已经统一
  - 卡片、阴影、圆角、字体和层次开始收束到同一设计语言

- [x] **标题卡开始承载栏目感**
  - TitleCard 已能展示主标题、日期和今日主题
  - 开场不再是纯品牌页

- [x] **字幕节奏更轻**
  - 标题卡、结尾卡和正文卡的字幕模式已区分

### 仍待补齐

- [ ] **全局品牌角标**
  - 需要在更稳定的位置展示 `HN TechPulse`
  - 建议加入日期和数据来源的固定角标

- [ ] **章节标识**
  - 每条 story 开头增加编号、主题标签和当前章节感
  - 让观众始终知道自己在看第几条、属于哪类议题

- [ ] **更统一的镜头节奏**
  - 让 Dashboard、Event、Atmosphere、Quote、Closing 的入场节奏更一致
  - 减少“有内容，但切得还不够像栏目”的感觉

### 验收标准

- 去掉文案后，截帧依然能看出这是同一栏目
- Story 切换清晰，观众不会丢失当前进度
- 动效是为了导视，不是为了装饰

## Milestone 4：数据解释与质量护栏

目标：降低自动化视频翻车概率，并提升指标可信度。

状态：进行中
优先级：P1

### 已完成项

- [x] **争议指数已有基础展示**
  - EventCard 已开始把争议分数转成更可读的中文标签
  - 争议不再只是一个孤立数字

- [x] **渲染前存在基础校验**
  - 缓存校验会检查 story 结构中是否包含 `editor_angle`、`why_it_matters`、`next_watch`
  - 这能避免部分半成品脚本直接进入渲染

- [x] **评论质量和观点分布已有数据入口**
  - 观点分布、关键词、评论质量分数已经能被提取和使用

### 仍待补齐

- [ ] **把争议指数解释完整**
  - 需要在画面上进一步解释为什么这条争议高
  - 可以增加评论密度、观点分散度或热度比等辅助说明

- [ ] **更完整的渲染前质量检查**
  - 标题长度、字幕行数、关键词数量、图片缺失、编码异常、时长异常都应统一检查
  - 失败时要自动降级，而不是只给 warning

- [ ] **事实来源与时间标注**
  - 每条视频应明确抓取日期和数据统计窗口
  - HN 分数、评论数、排名等指标要尽量保留原始来源字段

- [ ] **样片回归检查**
  - 需要一组固定 props / 历史数据样片做视觉回归
  - 至少覆盖长标题、无图、长评论和低评论量等常见边界情况

### 验收标准

- 渲染前能发现大多数潜在问题
- 新观众能看懂指标含义
- 样片能覆盖常见极端情况
- 降级后依然能产出可发布视频

## Milestone 5：表现力增强

目标：在不牺牲信息清晰度的前提下，提高完成度和观看愉悦感。

状态：待开始
优先级：P2

### 待办项

- [ ] **轻量音效和转场**
  - 只在开场、章节切换、指标准点处使用轻量音效

- [ ] **词云或关键词视觉化**
  - 作为 Atmosphere 的补充，而不是替代正文信息

- [ ] **趋势线 / 热度脉冲**
  - 把 score、comment_count、controversy_score 做成更直观的动态表达

- [ ] **更轻的角色化表达**
  - 若要加入人物化元素，必须非常克制，不能破坏专业感

### 验收标准

- 表现力是信息的放大器，不是噪音
- TTS 仍然清晰
- 新增元素可以通过配置关闭

## Milestone 6：发布与迭代闭环

目标：让 HN TechPulse 不只是能生成，而是能持续迭代。

状态：待开始
优先级：P2

### 待办项

- [ ] **定义成片评分表**
  - 统一评估开场吸引力、信息清晰度、观点价值、节奏、字幕可读性、数据可信度

- [ ] **A/B 测试不同栏目结构**
  - 例如 Top 3 深挖、Top 5 快扫、按主题分组三种结构

- [ ] **版本化 Prompt / 模板 / 样式**
  - 每次大改都能回滚到稳定版本

- [ ] **发布平台适配**
  - 横屏 16:9 先作为主版本
  - 未来再独立适配竖屏 9:16 和方形 1:1

### 验收标准

- 每次发布后知道哪里变好了、哪里变差了
- 栏目模板、Prompt 和样式都能回滚
- 不同平台能有各自合适的版式

## 推荐实施顺序

1. **先补 Milestone 3**：把栏目品牌和章节节奏做稳，让成片从“能看”变成“像一个固定栏目”。
2. **再推进 Milestone 4**：自动化越多，质量护栏越重要。
3. **之后再做 Milestone 5**：只有在信息表达稳定之后，增强表现力才有意义。
4. **最后补 Milestone 6**：把迭代、回滚和评估体系做成闭环。
