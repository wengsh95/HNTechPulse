# HN TechPulse Roadmap

> 最后更新：2026-05-11  
> 当前依据：已合入 `7b954ff Improve story enrichment and remotion rendering`，并对 Remotion 组件、文章 enrich、评论筛选和测试补充后的现状重新整理。

## 产品定位

HN TechPulse 是一个从 Hacker News 自动生成中文科技资讯视频的内容产品。它的目标不是把 HN 热榜简单搬运成摘要，而是做成一条有筛选、有判断、有节奏的技术社区观察栏目。

目标用户：

- 关注 AI、开发工具、开源、基础设施、隐私、安全和创业趋势的中文技术受众。
- 没时间每天刷 HN，但希望快速知道“今天技术圈在讨论什么、为什么重要、社区怎么分歧”的用户。
- 希望从 HN 评论里看到一线开发者、研究者和从业者真实反馈的用户。

核心价值：

- **节省时间**：把当天高价值讨论压缩成可观看的视频。
- **提供判断**：不只复述标题，还给出编辑角度、重要性解释和后续观察点。
- **保留社区温度**：呈现 HN 的真实分歧、观点分布和代表评论。
- **形成栏目感**：让观众一眼知道这是 HN TechPulse，而不是通用 AI 摘要视频。

## 当前状态

已具备的基础能力：

- HN 热榜与评论抓取。
- LLM 脚本生成，支持 JSON 解析重试、截断后自动扩展 token 上限和 segment 缓存版本控制。
- TTS 语音合成。
- Remotion 视频渲染，默认从 `public/props.json` 读取 props。
- Dashboard、Event、Atmosphere、Quote、Closing 等基础卡片。
- “事件详情 - 讨论气氛 - 代表观点”的 story 结构。
- 编辑角度、重要性、后续观察、观点分布、争议指数、代表评论等结构化字段。
- 开场页已能展示今日技术信号，而不只是品牌标题。
- 字幕已经区分标题卡、正文卡和轻量模式，减少部分视觉干扰。

最近已完成：

- 全局 `HN TechPulse` 品牌角标已加入正文段落，并显示日期/数据窗口。
- Story 章节编号已加入顶部 chrome，支持 `01/10` 这类进度信息。
- 底部进度条已加入 story 节点，并高亮当前 story。
- 开场主题更聚焦，示例 props 已改为“直接看今天的技术信号”。
- EventCard 已优化信息层级：编辑角度/中文标题优先、HN 原题弱化展示、关键词最多 3 个、关键点和“看点”分块。
- EventCard 已增加基础长文本保护：标题、摘要、关键点和标签都有 clamp/换行约束。
- 图片 enrich 已增加最小尺寸过滤、截图优先降级、Bing 查询数量控制和候选图片重排策略。
- AtmosphereCard 已把社区主情绪前置，观点分布压缩到前三类加“其他”，争议指数改为横向指标条并带解释标签。
- QuoteCard 已改为中文观点优先、一主两辅布局、立场标签贴近作者，并弱化英文原文。
- 评论筛选逻辑抽到 `comment_selection.py`，增加稳定 key、质量评分、引用过重惩罚和代表性选择。
- 新增/补充 `tests/test_article_enricher.py` 与 `tests/test_remotion_renderer.py`，覆盖 enrich 降级、图片选择和 Remotion props 输出的关键行为。

当前主要问题：

- 标准版时长还没有真正收敛到 `240-300s`，长视频仍需要内容结构层面的压缩。
- 标准版/完整版两种输出结构尚未形成稳定配置。
- EventCard 的来源感仍偏弱，目前主要展示 HN 原题，尚未稳定展示来源域名、HN id、分数和评论数。
- 图片选择已经改善，但仍需要质量检查和回归样片验证，避免搜索图或截图误导主题。
- 标题换行已有技术保护，但中文语义换行仍未系统处理。
- 代表观点页已减负，但字幕与卡片信息的配合还需要用真实成片继续校准。
- 渲染前质量护栏尚未建立，布局、图片、时长和乱码问题仍主要靠人工抽查。

## 北极星目标

让观众在前 10 秒知道今天最值得看的 3 个技术信号，并在每条 story 的 20-30 秒内理解三件事：

1. 发生了什么。
2. 为什么重要。
3. 社区真正分歧在哪里。

## Milestone 1：观看节奏与视频结构

目标：把视频从“完整扫读”优化成“可持续看完的技术日报”。

状态：待开始  
优先级：P0

### 待办项

- [ ] **控制标准版时长**
  - 标准版目标控制在 `240-300s`。
  - 开场约 `8s`，热点榜约 `10s`，Top 5 每条 `25-35s`，结尾约 `5s`。
  - Top 10 内容可以进入完整版或降级为一句话快扫。

- [ ] **支持标准版与完整版两种结构**
  - 标准版：更适合发布与日常观看，重点讲 Top 5。
  - 完整版：保留 Top 10，但必须加入章节、recap 和更强的节奏变化。

- [ ] **压缩单条 story 的表达**
  - 每条 story 建议使用三拍子：事件详情 `8-10s`、讨论气氛 `8-10s`、代表观点 `6-8s`。
  - 避免每条都展示过长摘要、长评论和完整图表。

- [ ] **增加中途 recap**
  - 完整版每 3-4 条后加入 3 秒小结。
  - 小结只呈现“到目前为止的共同主题”，避免重复标题。

### 验收标准

- 标准版总时长不超过 5 分钟。
- 单条 story 的核心信息能在 30 秒内讲完。
- 连续观看 3 条 story 后，画面和节奏不显得机械重复。

## Milestone 3：EventCard 新闻可信度与信息层级

目标：让事件详情页更像可信新闻解释，而不是泛科技视觉模板。

状态：进行中  
优先级：P0

### 已完成

- [x] **图片选择策略初版**
  - 已增加图片最小尺寸、候选排序、截图降级和 Bing 查询数量控制。
  - 已补充相关单元测试。

- [x] **基础标题和正文溢出保护**
  - 标题、摘要、关键点和标签已有 clamp 与换行保护。
  - 关键词数量已限制为 3 个。

- [x] **信息层级重排**
  - 编辑角度/中文标题优先作为主标题。
  - HN 原题弱化为辅助信息。
  - `key_points` 和 `why_it_matters` 被拆成更容易扫读的模块。

### 待办项

- [ ] **增强来源感**
  - 在 EventCard 中稳定展示来源域名或 HN story id。
  - 分数、评论数、抓取日期尽量保留，但不挤占主标题。

- [ ] **中文语义换行**
  - 当前已有溢出保护，但还没有按语义主动断行。
  - 长中文标题需要继续验证降字号、行宽和断句策略。

- [ ] **图片质量规则继续收敛**
  - 对 Bing 图或生成感强的图片增加更明确的过滤规则。
  - 对无图、低质图、小图建立可预测的无图布局或截图布局。

### 验收标准

- 每张 EventCard 在 2 秒内能看懂标题、来源和核心影响。
- 图片与新闻主题强相关，不出现明显“图库感”或误导性视觉。
- 长中文标题不撑坏布局。

## Milestone 4：讨论气氛与争议指数解释

目标：把社区分析做成产品差异化，而不是让用户读复杂图例。

状态：基本完成，待真实成片验收  
优先级：P1

- [x] **主情绪前置**
  - AtmosphereCard 已显示 `社区主情绪：支持/质疑/...`。

- [x] **简化观点分布图**

## Milestone 5：代表观点页减负

目标：把 QuoteCard 从“长评论展示”优化成“观点记忆点”。

状态：基本完成，待真实成片验收

  - 英文原文仅在主观点下弱化为短摘录。

## Milestone 6：视觉系统与动效节奏

目标：在不牺牲信息清晰度的前提下，提升完成度和观看愉悦感。

状态：进行中  
优先级：P2

### 已完成

- [x] **卡片质感初步收敛**
  - EventCard、AtmosphereCard、QuoteCard 的信息密度和层级已重新整理。
  - 主内容优先级高于装饰效果。

- [x] **模板入场节奏初版**
  - EventCard、AtmosphereCard、QuoteCard 已增加更有秩序的入场动画和逐项出现。

### 待办项

- [ ] **主题强调色系统**
  - 保留深色科技风，但按主题引入轻微色彩变化。
  - 隐私/控制权、AI、知识保存、安全、开发工具等主题可有不同强调色。

- [ ] **进一步收敛卡片视觉**
  - 继续检查圆角、阴影和边框是否过强。
  - 用抽帧对比判断是否更像新闻栏目，而不是泛科技模板。

- [ ] **轻量音效与转场**
  - 只在开场、章节切换、指标落点使用轻量音效。

### 验收标准

- 视觉变化服务于理解，而不是装饰。
- 连续观看 5 条 story 不感到模板过度重复。
- TTS 清晰度不被音效影响。

## Milestone 7：渲染前质量护栏与回归样片

目标：降低自动化视频翻车概率，让每天生成都可控。

状态：待开始  
优先级：P1

### 待办项

- [ ] **渲染前质量检查**
  - 检查标题长度、字幕行数、关键词数量、图片缺失、图片尺寸、编码异常、时长异常。
  - 检查 `story_scan` 实际时长是否超过目标时长。
  - 失败时给出自动降级方案，而不是只输出 warning。

- [ ] **图片质量检查**
  - 检测过小图片、纯色图、明显图标碎片、截图不可读等情况。
  - 图片质量不合格时自动回退到 screenshot 或无图布局。

- [ ] **固定回归样片集**
  - 至少覆盖长标题、无图、长评论、低评论量、高争议、10 条 story、中文乱码等情况。
  - 每次模板大改后抽帧对比。

- [ ] **成片评分表**
  - 统一评估开场吸引力、信息清晰度、观点价值、节奏、字幕可读性、数据可信度、视觉完成度。

### 验收标准

- 渲染前能发现大多数布局和数据问题。
- 降级后仍能产出可发布视频。
- 模板改动不会悄悄破坏历史样片。
