# HyperFrames Renderer (HN TechPulse)

Alternative video renderer that produces the daily HN briefing via HTML + GSAP
instead of Remotion. This directory is the **template** project — Python copies
and fills it on every `prepare_render` step into `data/{date}/hyperframes_project/`.

## Structure

```
hyperframes/
├── package.json            # minimal manifest; private, not published
├── index.html              # template; overwritten by Python
├── compositions/           # sub-compositions, one per card type
│   ├── cover.html          # opening card (tmp/openingcard.txt)
│   ├── event.html          # event card (tmp/eventcard.txt)
│   ├── atmosphere.html     # atmosphere card (tmp/atmospherecard.txt)
│   └── closing.html        # closing card (tmp/closingcard.txt)
└── public/                 # populated by write_props() (audio + images)
```

## CLI workflow

The Python driver (`uv run python main.py --renderer hyperframes --steps render`)
handles the full pipeline. To drive `npx hyperframes` directly inside a
generated project:

```bash
cd data/2026-06-07/hyperframes_project
npx hyperframes lint
npx hyperframes inspect --samples 10
npx hyperframes preview --port 3002
npx hyperframes render --output final.mp4 --quality standard
```

## Variables

Each sub-composition declares its `data-composition-variables`. The root
`index.html` overrides them per instance via `data-variable-values='{...}'`.
That JSON is generated by `hyperframes_props.py`.

## Constraints

- Node >= 22, FFmpeg, Chrome (HyperFrames downloads its own Chromium on first run)
- Each scene's audio path must resolve to a file under `public/audio/`
- Image paths must resolve under `public/images/`
