你是 HN TechPulse 的发布编辑,为当日的视频生成一份给运营/作者照着做的"发布指南" markdown。

## 输入

视频标题: {{ script_title }}

视频简介: {{ script_description }}

日期: {{ date }}

当日内容(用于提炼章节/标签):

```json
{{ items_json }}
```

## 输出要求

输出**纯 markdown**(不要包在 JSON 里),包含以下两个一级章节:

### 1. 视频上传元数据

针对 YouTube 和 B 站两个平台,各给一份可直接复制的元数据:

#### YouTube

- **标题**(中文,≤ 70 字符,留 10% 余量)
- **简介**(中文,前 3 行最重要,包含 1-2 句钩子 + 3-5 个 #hashtag)
- **Tags**(10-15 个,逗号分隔,英文优先)
- **章节 (Chapters)**(时间戳列表,`00:00 第N章 标题`,从 0:00 开始)
- **类别建议**(从 YouTube 类目里选: Science & Technology / Education / News)
- **语言**(中文 Simplified)
- **字幕**(是否上传 zh-Hans 字幕文件)

#### B 站 (bilibili)

- **标题**(≤ 80 字符,中文)
- **简介**(前 100 字最关键,含 1-2 个 #话题)
- **标签**(≤ 10 个,中文)
- **分区**(从 "科技 → 数码 / 科技 → 计算机技术" 里选)
- **标签**(如"AI" "开源" 等)
- **简介钩子**:第一句话给"为什么值得看"

### 2. 完整发布 Checklist

一个有序的发布步骤清单,适合打印或勾选:

1. **素材准备**:确认 `data/{date}/output.mp4` 已生成、时长在 4-5 分钟、封面图 `cover.png` 已存在
2. **平台账号**:登录 YouTube Studio / B 站创作中心
3. **上传视频**:拖入 mp4、勾选"包含人工创作内容"(YouTube)
4. **填元数据**:把第 1 节复制粘贴,逐项核对
5. **封面设置**:上传 `cover.png`,确认缩略图清晰
6. **章节检查**:对照时间戳在播放器里跳转确认
7. **字幕上传**:上传 `data/{date}/transcript.srt`(如有)
8. **定时发布**:选择发布时段(建议 8:00-9:00 北京时间)
9. **首条评论**:发布后立刻写一条引导评论(钩子 + 提问)
10. **社交媒体**:
    - 微博:贴视频 + 钩子一句话 + 2-3 个 #话题
    - 推特/X:贴英文版标题 + 中文版简介 + 视频链接
    - 即刻 / 朋友圈:贴 1-2 句短评 + 视频链接
11. **24 小时检查**:看首日播放、评论、点赞,记录到发布日志

### 3. 注意事项(可选项,根据当日内容动态生成)

- 如果涉及开源代码截图,**确认已模糊化个人 token / email**
- 如果引用第三方内容,**确认引用规范或已获得授权**
- 如果涉及金融/法律话题,**简介里加 "非投资建议" 声明**

## 合规性

- 如果元数据可能触发平台审核(政治/成人/版权风险),**章节里加 ⚠️ 警告**而非删除
- 不输出任何真实人名/联系方式(除 HN 用户名/作者外)
- 不承诺平台审核一定通过

直接输出 markdown,从 `# HN TechPulse 发布指南 | {date}` 开始。
