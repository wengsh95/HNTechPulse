## 输出 JSON 结构

严格输出 JSON，不要输出其他文字。

```json
{
  "article_summary": "150-300字中文摘要",
  "editor_angle": "10-18字编辑提炼句",
  "dek": "32-55字背景脉络",
  "key_points": [
    {"label": "为何关注", "text": "18-34字"},
    {"label": "影响", "text": "18-34字"}
  ],
  "keywords": ["关键词1", "关键词2", "关键词3"],
  "category": "AI工具|开源生态|基础设施|安全风险|创业信号|开发者体验|硬件与系统|其他",
  "why_it_matters": "为什么这条新闻值得关注，14-24字"
}
```

## 字段要求

### article_summary

- 150-300字中文
- 覆盖文章核心内容、关键结论和重要数据
- 不要复述标题，提供标题之外的信息增量
- 如果文章包含代码/技术细节，简要概括其技术方案

### editor_angle

- 10-18个中文字符，主谓宾结构
- 像视频卡片标题，不像文章标题
- 必须是完整短句，不能为了凑长度截掉后半截；宁可更短，也不要出现“可复制的提”“立法机”这类半截词
- 突出关键事实差异或技术信号
- 不要句号，不要冒号，不要照翻英文标题
- 不要无主语的空话，如"XX引发关注""XX迎来变革"
- 修辞词（"另一面""重塑""风暴"等）慎用，**用了必须有具体事实兜底** —— 100% 干瘪口播和 100% 修辞腔都不可取
- **受众是 HN / 技术 / 创投圈读者**, 假设他们熟知主流公司、产品、模型、项目名 (Anthropic, OpenAI, Google/Alphabet, Meta, DeepMind, Stripe, Cloudflare, AWS, NVIDIA, Cursor, etc.)
- **公司/产品/项目/模型名一律保留原文, 不要用"AI 公司""芯片巨头"这类通用描述替换** (不写"AI 融资狂飙公司递交招股书"代替"Anthropic 递表")
- 直接说谁做了什么, 不为圈外解释背景
- 不要修改事实，不要捏造事实

### dek

- 32-55个中文字符
- 补充 editor_angle 未涵盖的背景脉络或来龙去脉
- 必须忠于原始素材的事实和措辞，不得对事实做任何无依据改写
- 不要与 key_points 重复同一句事实

### key_points

- 固定2条
- key_points[0].label 固定为"为何关注"，key_points[1].label 固定为"影响"
- key_points[].text：18-34个中文字符，只写一个具体信息点
- 为何关注：回答它为什么值得技术人停下来判断，突出趋势信号、异常变化或价值差异
- 影响：回答波及谁、影响什么范围

### keywords

- 固定3个，用于视频卡片底部标签和正文高亮
- 每个关键词应是“可显示、可高亮、信息密度高”的短标签
- 优先选择文章中的产品名、项目名、协议名、模型名、公司名、技术机制或具体风险点
- 允许中英文混排，例如 "Claude Code"、"MCP协议"、"本地推理"
- 每个关键词建议 2-8 个中文字符，或 1-3 个英文词；不要超过 14 个字符
- 三个关键词尽量分工：核心对象、核心变化、影响范围
- 核心对象：产品 / 项目 / 技术名
- 核心变化：发布 / 开源 / 降价 / 漏洞 / 性能提升 / 架构调整
- 影响范围：开发者体验 / 云成本 / 安全风险 / 开源生态 / 企业部署
- 不要输出泛词：技术、工具、平台、系统、应用、创新、趋势、生态、人工智能
- 不要和 category 完全重复
- 尽量选择会出现在 dek、why_it_matters 或 key_points 中的词，便于视频高亮

### category

- 从枚举中选择一个最贴切的分类：AI工具、开源生态、基础设施、安全风险、创业信号、开发者体验、硬件与系统、其他

### why_it_matters

- 14-24个中文字符
- 回答"为什么技术人应该关心这条新闻"：它改变了什么、暴露了什么趋势、或者动摇了什么已有认知
- 必须是完整判断句，不要像标题一样压缩到读不顺；不要用“重塑”“变革”这类空泛词独立成句
- 不要重复 editor_angle 已经说过的具体事实
- 面向技术读者，可以点出技术层面的信号意义
- 如果有搜索结果作为上下文，参考其补充信息
