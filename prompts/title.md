## 要求

### title(三个关键词格式,必出 3 候选)

- **核心格式:三个并排的「关键词」**,用「、」分隔,**总字数 18-24 字**(硬上限 28 字)。每个关键词 6-8 字。
- B 站缩略图只有 3 行 × 14 字宽度,封面副文会单独走 CoverThumbnail 组件叠加,标题本身要短促有力。
- 三个关键词必须直接命名当日 3 个核心话题/事件的具体冲突点或结果,不要用抽象标签("AI 资讯"、"今日速览"这种一概禁止)。
- **三个钩子要有差异化**——主语对比 / 反差强度 / 数据密度三选一,便于用户挑选。
- **新用户视角自检**:写完每个候选都要问自己——没看过原稿的 B 站用户扫一眼 0.3 秒,能不能秒懂在讲啥?三个钩子不能都是行话压缩("端侧部署"、"KV 缓存"这种),至少有一个是结果/数字/反常识这种秒懂钩子。
- **真实数字硬约束**:数字必须真实,可从 highlight_entries 里读 HN 分数/评论数/具体数据;**不准编造**"周下载 1.29 亿"这种幻觉数字——用户会核对。
- 风格参考(已经收敛过的方向):
  - "Gemma 4 压进 1GB、Transformer 砍一刀投影、千人晒被 AI 颠覆" (29 字 → 可压到 24 字)
  - "开源模型塞进 1GB、注意力投影两组就够、737 条评论被打脸" (28 字)
  - "端侧大模型 1GB、QKV 投影被砍、HN 千人复盘被反杀" (24 字)
- **不**使用 emoji、不使用「【」/!/? 标点
- **不**在标题里塞英文人名,改用中文译名或代称("奇幻作家"等);**公司/产品名**(Gemma、Claude、Anthropic、Google、Cloudflare 等)可保留英文
- **必须输出 3 个标题候选**,放在 `title_candidates` 数组,候选之间有差异化(主语对比 / 反差强度 / 数据密度)
- 第一个候选 = 推荐主推,放进 `title` 字段(LLM 选定的 best pick)
- **每个候选长度差 ≤ 5 字**(避免 14/24/35 这种长度不均)

### description
- 中文,80-120 字(B 站简介 + YouTube 简介复用)
- 概述今日讨论的 2-3 个核心主题,带具体故事名/概念
- 结尾给一句"为什么值得看"的钩子

### cover_subtitle(封面副文,专门给封面图用)
- 中文,**20-40 字,严格上限 40 字**——封面缩略图只有 320x180,长文必糊
- 3 个短促的破折号短语,每条不超过 12 字
- 形式:`破折号 + 一个数据或钩子`,让缩略图缩小后还能读
- 不用句号、不用完整句,直接 3 个并列短语

### tags
- 5-8 个中文标签
- 优先用主题分类(AI / Infra / 开源 / 安全 / 隐私 / 开发工具),而非具体产品名
- 不超过 2 个具体公司/产品标签

## 合规
- 政治敏感、暴力、色情、仇恨言论、个人隐私泄露 → **输出空 title / description / tags 数组**(调用方会兜底默认)
- 不输出可能触发内容审核的表述

只输出 JSON,不要其他文字。
