## 要求

### title(三段式信息密度优先,必出 3 候选)
- **核心格式:三段并列 / 三段递进 / 三段反转,用"、"分隔,总字数 25-35 字**(硬上限 40 字)—— 2026-06-06 起 CoverThumbnail 组件支持 `\n` 显式换行,B 站缩略图下 3 行 14 字不糊
- 每段 6-12 字,直接命名当日 3 个核心话题/事件的具体冲突点或结果(关键词太抽象,新用户看不懂—— 2026-06-06 反馈)
- 第三段常是反转/争议/代价那一击(跟前两段形成递进或反钩)
- 避免 "今日 HN 速览" 这类无信息量总结
- 风格参考(实际在用):
  - 具体结果递进(19 字): "Gemma 4 跑进 1GB、缓存压缩 97%、韩国审查"
  - 三段冲突并列(20 字): "权重写科幻、AI 写代码、UC 不及格率破三成"
  - 反差钩子(20 字): "AI 用十秒改完人类十分钟的科幻,伯克利却因 AI 挂科率飙到 35%"
  - 数字冲击并列(20 字): "1397 票 AI 元叙事,35% 伯克利挂科,Vite 团队一夜入 Cloudflare"
- 不要使用 emoji、不要使用 "【】" "!" "?" 标点
- 不要在标题里塞英文人名(用中文译名或"奇幻作家"等代称);公司/产品名(Gemma、Claude、Anthropic、Google)可以保留英文
- 数字必须真实(可从 `{{ highlight_entries }}` 里读 HN 分数/评论数/具体数据;**不准编**"周下载 1.29 亿"这种幻觉数字—— 用户会核)
- **必须输出 3 个标题候选,放在 `title_candidates` 数组里**,3 个之间应有差异化(主语对比 / 反差强度 / 数据密度),便于用户挑选
- 第一个候选 = 推荐的 `title` 字段(代表 LLM 选定的 best pick)
- **每个候选长度差 ≤ 5 字**(避免 10/25/36 这种长度不均)
- **新用户视角自检** (2026-06-06 加入): 写完每个候选都要问自己——"没看过原稿的 B 站用户扫一眼 0.3 秒,能不能秒懂在讲啥?" 三个钩子不能都是行话压缩("端侧部署"、"KV 缓存"这种),至少有一段是**结果/数字/反常识**这种秒懂钩子

### description
- 中文,80-120 字(B 站简介 + YouTube 简介复用)
- 概述今日讨论的 2-3 个核心主题,带具体故事名/概念
- 结尾给一个 "为什么值得看" 的钩子

### cover_subtitle(封面副文,专门给封面图用)
- 中文,**20-40 字,严格上限 40 字**——封面缩略图只有 320x180,长文必糊
- 3 个短促的破折号短语,每条不超过 12 字,例如:
  - "—— 80% 代码由 Claude 写"
  - "—— 漏洞管道五天上手"
  - "—— 17% 漏放率争议"
- 形式:`破折号 + 一个数据或钩子`,让缩略图下还能读
- 不要用句号结尾,不要用完整句子,直接 3 个并列短句

### tags
- 5-8 个中文标签
- 优先用主题分类(AI / Infra / 开源 / 安全 / 隐私 / 开发工具)而非具体产品名
- 不要超过 2 个具体公司/产品名

## 合规性

- 如果涉及政治敏感、暴力、色情、仇恨言论、个人隐私泄露,**输出空 title 空 description 空 tags 数组**(调用方会兜底默认值)
- 不输出可能触发内容审核的表述

只输出 JSON,不要其他文字。
