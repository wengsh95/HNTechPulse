## 要求

### title(必出 3 候选)

**长度约束:**
- 标题总长度控制在 **30 字以内**(含标点和英文)。英文产品名(Gemma、Claude、Transformer 等)按视觉宽度算,一个英文单词约等于 2 个中文字。
- 判断标准:在 B 站视频标题栏**一行能放下**、不被截断,就够了。不要逐字符计数,目测判断即可。

**格式与风格(3 个候选必须用不同风格):**
- **候选 A — 三段式**: 用「、」分隔三个钩子,每个钩子点一条故事的核心矛盾。例:`Claude 改写自己、Agent 沙箱拆解、漏洞猎手开源`
- **候选 B — 主线聚焦**: 只抓当天最有爆点的一条故事做主标题(10-15 字),其余用"还有:XX"一笔带过。例:`十年老 Xeon 跑通大模型,还有 N 卡杀入 ARM 笔电`
- **候选 C — 悬念/反常识**: 用疑问、反差或数据做钩子,制造信息缺口。例:`HN 千人讨论:GPU 空转率竟高达 59%`

**内容约束:**
- 直接用当日故事的具体冲突点或结果,不要用抽象标签("AI 资讯"、"今日速览"一概禁止)。
- 数字必须来自 highlight_entries,不准编造。
- 至少有一个钩子是"秒懂型"——不看原稿也能立刻产生好奇心。
- 不使用 emoji、不使用「【」/!/? 标点。
- 英文人名翻译成中文或省略;公司/产品名(Gemma、Claude、Anthropic、Google 等)可保留英文。

**输出规则:**
- 3 个候选放 `title_candidates` 数组。
- 第一个候选 = 你认为最好的,同时放进 `title` 字段。
- 3 个候选长度大致接近(差距不超过 10 字)。

### description
- 中文,80-120 字(B 站简介 + YouTube 简介复用)
- 概述今日讨论的 2-3 个核心主题,带具体故事名/概念
- 结尾给一句"为什么值得看"的钩子

### cover_subtitle(封面副文,专门给封面图用)
- 中文,**20-40 字,严格上限 40 字**——封面缩略图只有 320x180,长文必糊
- 3 个短促的破折号短语,每条不超过 12 字
- 形式:`破折号 + 一个数据或钩子`,让缩略图缩小后还能读
- 不用句号、不用完整句,直接 3 个并列短语

### tags
- 5-8 个中文标签
- 优先用主题分类(AI / Infra / 开源 / 安全 / 隐私 / 开发工具),而非具体产品名
- 不超过 2 个具体公司/产品标签

## 合规
- 政治敏感、暴力、色情、仇恨言论、个人隐私泄露 → **输出空 title / description / tags 数组**(调用方会兜底默认)
- 不输出可能触发内容审核的表述

直接输出 JSON,不要在 thinking 中逐字符计数。
